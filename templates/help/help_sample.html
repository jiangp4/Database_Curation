Sample annotation table

In the sample annotation table, you will label a few destination columns to enable automatic analysis by computer programs. Each row represents one sample in the current experiment. Please review all meta information columns and create the destination columns in standardized vocabularies. You can either manually input the values or utilize the assist functions provided below. Finally, a computer program should be able to analyze this dataset automatically through the destination columns (together with the genomics data processed in a separate process).

Many features in the sample annotation table are the same as those in the dataset selection table in Stage one, such as the column filters, progress bars, scroll bars, table headers, and result submission. Therefore, the following functions are unique to the annotation functions in the current stage.



  1. Column selection. All assist functions are targeting a Destination Column. Functions, including Copy (2a) and Join (7), will directly involve a Source Column.
  2. Copy content to the destination column.
    • From the source column
    • From the Clipboard, triggered by Ctrl + V or Command + V (Mac OS). You can either paste a whole column copied from an Excel table or a text content to all cells in the destination column.
  3. Clear all content in the destination column.
  4. Delete the destination column.
  5. Add a new destination column.
  6. Undo or Redo annotations from the assist functions.
  7. Join the source column to the front or back of the destination column with the separator in the text input box.
  8. Append the word in the text input box to the front or back of the destination column.
  9. Split a substring from the front or back of the destination column. In each cell, the content will be split by the text in the input box. Advanced users can input regular expressions by using “/pattern/” or “/pattern/i” for case insensitive queries.
  10. Replace substrings (text input box “From”) in the destination columns to a target string (text input box “To”). Advanced users can input regular expressions by using “/pattern/” or “/pattern/i” for case insensitive queries.
  11. Translate contents to standardized vocabularies in the destination column. In the project setting, the user may define a vocabulary map from non-standard texts to their synonyms in controlled vocabularies. Be cautious that the automatic translation might misinterpret the content, and human proofreading is always necessary.
